Nutrition Analysis Application (GitHub)

The "Nutrition Analysis Application" is an innovative AI-driven web application that provides detailed nutritional information for various foods. The user simply uploads an image of a dish, and the application does the rest. Leveraging advanced Machine Learning (ML) techniques, the application recognizes and classifies the food items in the image, estimates their volume, and then consults the NutritionIX API to retrieve comprehensive nutritional data. This allows the user to easily track the nutritional value of their meals, promoting healthier eating habits and lifestyle choices.

This application is composed of two main parts: the client-side user interface and the server-side data processing unit. The client-side application is built using React, which facilitates a smooth and intuitive user experience. The server-side, written in JavaScript using Node.JS, processes the image, interacts with the ML model, retrieves data from the NutritionIX API, and sends the results back to the client.

    Skills and Technologies Utilized
  1. Machine Learning: Utilized deep learning methods for food classification and volume estimation. This involved training a model with a diverse dataset of food images.
  2. Python and Flask: Developed the model APIs using Python and Flask, handling image processing tasks, API interactions, and data transfer operations.
  3. Node.js: Built part the server-side application using Node.js, which provided the infrastructure for deploying the machine learning model and communicating with the client-side application.
  4. ReactJS: Built a user-friendly front-end application with ReactJS, providing users with a clean, intuitive interface for uploading images and viewing nutritional data.
  5. NutritionIX API: Leveraged the NutritionIX API to fetch detailed nutritional data for identified food items.
  6. React Bootstrap: Utilized React Bootstrap to enhance the design and responsiveness of the application, ensuring a pleasing aesthetic and smooth user experience across different device types.
  7. Data Analysis: Performed detailed data analysis for nutritional data, enhancing user understanding of their dietary intake.

What I learned:

This project was a comprehensive exercise in full-stack development, machine learning, and data analysis. It provided valuable experience in integrating front-end and back-end technologies to create a cohesive and functional application. The use of machine learning models for image recognition and volume estimation presented both challenges and learning opportunities, strengthening my understanding of these technologies. Furthermore, the process of fetching and manipulating data from the NutritionIX API expanded my skills in working with external APIs and performing detailed data analysis.

Overall, this project has significantly improved my technical skills, problem-solving abilities, and understanding of the potential applications of AI in the health and nutrition sector.

Track Expenses (Android) (GitHub)

TrackExpenses is an Android application which allows the user to add, edit, or delete any of the expenses that are displayed using a RecyclerView (expenses are saved in the internal storage of the device as a .txt file).
The project contains a main activity which serves as a fragment container, and two fragments - one for displaying the expenses and another for adding or editing expenses.
In addition, the project is built using MVVM architecture for separating the view and the logic (the adapter acts as the Model, the fragment for displaying the recyclerview is the View, and the View-Model holds the live data).
The menu contains the following functionalities:

  • Add a new expense.
  • Sum of current displayed expenses button.
  • Filter expenses by date. The user has the ability to choose the year, year-month, or leave both fields empty to display all expenses ever added.
  • Sort expenses by name, date, or amount. The chosen sorting method is saved using SharedPreferences to allow the expenses to be displayed by that order upon the next launch of the app.

A broadcast receiver is used when launching the app, which broadcasts the current month and year, thus, displaying the relevant expenses. Lastly, a foreground service is used, which uses another broadcast receiver that sends a daily notifiction to the user at 8 PM as a reminder to enter their daily expenses.
